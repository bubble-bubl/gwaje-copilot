과제 공지 분석기 프로젝트 인수인계 문서
현재 상태: 1·2단계 완료 / 다음 작업: 3단계 ICS 일정 다운로드
배포 주소	https://gwaje-copilot-44.streamlit.app/
현재 스택	Streamlit / Gemini 2.5 Flash / Supabase / GitHub 배포
진행 단계	1단계 최근 분석 보기 완료, 2단계 프롬프트 종류 추가 완료

1. 프로젝트 개요
이 프로젝트는 대학생이 교수 공지, 과제 안내문, LMS 공지, 캡처 이미지를 입력하면 내용을 빠르게 해석해 즉시 행동 가능한 형태로 정리해주는 웹앱을 만드는 것이 목적이다.
이 앱은 범용 AI 챗봇이 아니라 대학생이 공지를 처리할 때 더 빠르고 편하게 쓰는 전용 도구로 설계되어 있다.
2. 제품 방향 및 포지셔닝
•	긴 공지를 짧고 실용적으로 정리
•	사용자가 실제로 해야 할 행동을 추출
•	GPT·Gemini에 다시 설명하는 번거로움 제거
•	일정 등록까지 이어질 수 있는 구조 마련
핵심 포지션은 ‘범용 AI 앱’이 아니라 ‘대학생 과제 처리 전용 워크플로우 도구’이다.
3. 현재 기술 스택 및 운영 환경
•	프론트/앱 프레임워크: Streamlit
•	AI 모델: Gemini 2.5 Flash
•	데이터베이스: Supabase
•	버전 관리/배포: GitHub 연동 + Streamlit Cloud
•	현재 배포 주소: https://gwaje-copilot-44.streamlit.app/
•	Gemini API, Supabase API, GitHub 배포 모두 연결 완료
4. 현재까지 구현 완료된 기능
4-1. 입력 기능
•	텍스트 입력: 공지문 직접 붙여넣기
•	이미지 업로드: 공지 캡처 이미지 업로드
4-2. AI 분석 기능
Gemini 2.5 Flash로 공지 내용을 분석하여 JSON 형태로 구조화한다.
•	핵심 요약
•	마감일
•	해야 할 일
•	제출물
•	주의사항
•	AI용 프롬프트
•	일정 등록용 문구
4-3. 파싱 안정화
•	response_mime_type='application/json' 적용
•	temperature=0.1로 조정
•	코드블록 제거 및 JSON 정리 함수 적용
•	파싱 실패 시 원본 응답 디버깅 가능
4-4. UI 구조 정리
초기에는 항목이 많아 난잡했으나, 현재는 아래 구조로 정리 완료.
•	핵심 요약
•	마감일
•	해야 할 일
•	제출물
•	주의사항
•	AI용 프롬프트
•	일정 등록용 문구
통합/제거 사항:
•	과제명 → 핵심 요약에 녹임
•	제출 형식 → 제출물에 합침
•	준비물 / 참고 요소 → 제거
•	GPT용 / Gemini용 프롬프트 분리 → 단일 AI용 프롬프트 카드로 통합
•	결과 영역은 expander 방식으로 접기/펼치기 가능
4-5. 최근 분석 보기 기능
•	Supabase에 저장된 최근 분석 기록 5개 표시
•	저장 시각, 마감일, 핵심 요약, 원본 JSON 보기 제공
•	created_at은 YYYY-MM-DD HH:MM 형식으로 포맷팅
4-6. 분석 결과 유지 기능
•	st.session_state 사용
•	프롬프트 종류 변경 시 분석 결과 유지
•	selectbox 변경으로 인한 화면 초기화 문제 해결
4-7. AI용 프롬프트 기능
분석 결과를 바탕으로 목적별 프롬프트 선택 가능.
•	자세한 설명용
•	보고서 / 과제 초안용
•	발표 준비용
•	팀플 역할분담용
의도: AI를 대신하는 것이 아니라 사용자가 GPT/Gemini에 바로 넣어 작업을 이어갈 수 있도록 돕는 것.
5. 현재 UI 및 기능 방향의 확정 사항
•	과제명은 별도 카드로 두지 않고 핵심 요약에 녹인다.
•	제출 형식은 제출물에 합친다.
•	준비물 / 참고 요소는 제거한다.
•	주의사항은 유지한다.
•	AI 프롬프트는 하나의 카드 안에서 다룬다.
•	일정 등록용 문구는 향후 캘린더 연동을 위해 유지한다.
6. 지금까지 해결된 문제
•	모바일 헤더 줄바꿈 이슈
•	JSON 파싱 실패 빈도 감소
•	결과 항목 과다 문제 정리
•	expander 제목 색상 문제 수정
•	selectbox 변경 시 결과 초기화 문제 해결
•	최근 분석 보기 구현 완료
•	저장 시각 포맷 가독성 개선
7. Supabase 관련 상태
현재 사용 테이블: usage_logs
•	주요 컬럼: id, created_at, input, output, input_type, parsed_json
•	텍스트 분석 결과 저장 정상
•	최근 분석 보기에서 불러오기 정상
8. 개발 순서와 현재 진행 상태
이 프로젝트는 아래 순서를 고정해서 진행하기로 이미 확정했다.
1.	최근 분석 보기
2.	프롬프트 종류 추가
3.	ICS 일정 다운로드
4.	이후 캘린더 연동 검토
현재 진행 상태: 1단계 완료, 2단계 완료, 현재부터 3단계 시작 가능.
9. 다음 단계에서 바로 해야 할 일
9-1. 3단계 목표: ICS 일정 다운로드 기능 추가
현재 앱에는 일정 등록용 문구가 생성되지만 아직 텍스트 수준이다. 다음 단계에서는 사용자가 일정 앱 또는 캘린더에 바로 넣을 수 있는 .ics 파일을 생성하고 다운로드할 수 있게 만들어야 한다.
9-2. 3단계 구현 목표
•	분석 결과의 마감일을 기반으로 일정 데이터 생성
•	.ics 형식의 캘린더 파일 생성
•	Streamlit에서 다운로드 버튼 제공
•	날짜 파싱 실패 시 예외 처리
•	일정 제목은 summary 또는 적절한 축약 정보 기반으로 생성
9-3. 3단계에서 고려할 점
•	마감일이 정확한 날짜/시간 형태가 아닐 수 있음
•	‘공지 확인 필요’ 값일 때 ICS 생성 제한 필요
•	한국 시간 기준 일정 파일 처리 여부 결정 필요
•	일정 제목이 너무 길지 않도록 정리 필요
10. 이후 단계
3단계가 끝나면 4단계인 캘린더 연동 검토로 이동한다. 고려 가능한 확장 예시는 Google Calendar 링크 생성, 실제 캘린더 연동, 일정 수정/삭제 구조 검토 등이다. 단, 지금은 3단계 ICS 일정 다운로드부터 우선 처리해야 한다.
